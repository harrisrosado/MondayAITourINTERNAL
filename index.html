🚨 SE Positioning: Sidekick vs. MCP (Claude / Copilot)
Sidekick (position this)
• Lives inside monday — sees boards, items, people
• Takes actions: creates items, updates statuses, builds workflows
• Uses monday AI credits (predictable, admin-controlled)
• Zero setup on Standard+ plans — works instantly in-platform
MCP to Claude / Copilot (be careful)
• Lives outside monday — reads data via API connector
• Can query but can't natively act inside monday
• Uses customer's own LLM license (bypasses monday credits)
• Great for power users, but cannibalizes Sidekick adoption
The talk track: "MCP is great for devs who live in their IDE. But for the 90% of your org that works in monday every day, Sidekick is the answer — no setup, full context, takes action. Don't let the tail wag the dog."

What makes a great Vibe

✅ Do this
• Start with the customer's pain, not a feature list
• Describe the persona who will use it daily
• Specify the data it needs to connect to
• Mention the actions users should be able to take
• Include visual preferences (dark mode, dashboard style, card layout)
• Iterate — round 1 is never the final product
❌ Not this
• "Build me an app" — too vague, generic output
• Feature shopping — listing 20 things without context
• Ignoring the user — who actually uses this thing?
• Skipping data connections — an island app is useless
• One-and-done prompting — always refine
• Forgetting governance — who can publish? who can see it?

Part 4

Translating pain into Vibe

From customer problem to custom app in minutes

The framework

The Vibe prompt formula

P
Pain
What's broken? Use the customer's exact words.
P
Persona
Who uses this app every day? What's their role?
D
Data
What boards, columns, or external data should it connect to?
A
Actions
What should users be able to do? Filter, approve, escalate, assign?
V
Visual
Dashboard? Card layout? Dark mode? How should it look and feel?
P.P.D.A.V. — Pain, Persona, Data, Actions, Visual
A Vibe prompt with all five is 10x better than "build me an app."

Same request, different prompt

❌ Lazy prompt
"Build me an app for tracking quality issues in our factory"
Result: Generic form + list. No context, no actions, no data connection. Useless in a demo.
✅ PPDAV prompt
"Pain: Our QC team logs defects on paper, takes 3 days to spot trends.
Persona: Floor supervisors who aren't tech-savvy, use iPads.
Data: Connect to our Production Tracking board (SKU, line, shift columns).
Actions: Log defect with photo, auto-categorize type, alert manager if critical, weekly trend chart.
Visual: Mobile-first card layout, big buttons, dark mode for factory floor."
Result: A demo-ready app the customer can see themselves using tomorrow.
